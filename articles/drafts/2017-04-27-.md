# 2. The fast way

Another approach is to consider all observed letters as a big urn. Since letters where more often observed are to find more often in this urn  a uniform random draw from this “urn” then mimics the true distribution \_\_WPMATH\_0000\_\_. This Idea is quite similar to the idea of [Bootstrapping](https://en.wikipedia.org/wiki/Bootstrapping_%28statistics%29). A big advantage of this method is that it is really fast since we do not need an estimator \_\_WPMATH\_0001\_\_ for \_\_WPMATH\_0002\_\_ which will be computationally costly. The same is obviously also true for the password length, i.e. first sample a length from the observed set (“length urn”) to mimic \_\_WPMATH\_0003\_\_ and then sample \_\_WPMATH\_0004\_\_ times from the “letter urn”.

This proedure also works for the markov chains, but we need a lot of urns then. Consider a chain of order \_\_WPMATH\_0005\_\_ we need \_\_WPMATH\_0006\_\_ urns, for each observation one urn to continue the chain. To introduce the “smoothing” we add one element of the unobserved quantities to the urn.
